## 1. 项目概述

### 1.1 项目名称

**CreditDemand-FE** — 基于征信报告数据的用户需求预测特征工程项目

### 1.2 一句话描述

从个人/企业征信报告原始数据中，系统化构建高质量特征，训练并评估用户需求预测模型，为精准营销、产品推荐与风控决策提供数据支撑。

### 1.3 业务背景

金融机构在获客、授信、产品推荐等环节，需要理解用户的真实金融需求。征信报告是权威、结构化的信用信息载体，包含信贷历史、还款行为、查询记录、负债结构等丰富信号。本项目目标是将这些非结构化/半结构化征信数据转化为可建模的特征体系，并预测用户的潜在需求（如：贷款品类偏好、额度需求区间、申请意向概率等）。

### 1.4 业务价值

| 价值维度 | 说明 |
|---------|------|
| 精准营销 | 基于征信特征识别高意向用户，提升转化率 |
| 产品匹配 | 预测用户更适合的贷款/信用卡/理财产品类型 |
| 风险与需求联动 | 在可承受风险范围内推荐合适额度与期限 |
| 可解释决策 | 特征工程产出可审计、可复现，满足合规要求 |

## 2. 目标与范围

### 2.1 核心目标

1. **数据理解**：完成征信报告原始数据的字段梳理、质量诊断与样本定义
2. **特征工程**：设计并实现覆盖多维度征信信息的特征体系（目标 200+ 基础衍生特征，可扩展至 500+）
3. **标签构建**：定义并生成用户需求预测标签（见 2.3）
4. **建模基线**：建立可复现的训练/评估流水线，输出基线模型与特征重要性报告
5. **文档与交付**：产出特征字典、数据血缘、实验记录与部署就绪的特征计算脚本

### 2.2 项目范围（In Scope）

- 征信报告解析后的结构化/半结构化数据
- 离线特征工程流水线（批处理）
- 探索性数据分析（EDA）与特征有效性评估（IV、PSI、相关性等）
- 基线预测模型（逻辑回归 / LightGBM / XGBoost）
- 特征可解释性分析（SHAP / 特征重要性）

### 2.3 项目范围（Out of Scope — 首期不做）

- 实时在线推理服务（API 部署）— 可作为二期
- 非征信类替代数据源（社交、消费行为等）— 除非业务方明确要求
- 生产级 MLOps 全链路（Airflow/Kubeflow）— 首期以脚本化流水线为主

### 2.4 预测目标定义（待根据实际业务确认）

请在接入数据后，从下列选项中**明确一种或多种**作为建模标签：

| 标签类型 | 定义示例 | 任务类型 |
|---------|---------|---------|
| **需求品类** | 用户未来 30 天内申请的产品类型（消费贷/房贷/信用卡/…） | 多分类 |
| **需求意向** | 用户未来 N 天内是否有明确申请行为（是/否） | 二分类 |
| **额度需求** | 用户期望或可承受额度区间（如 <5万 / 5-20万 / >20万） | 多分类或回归 |

### 3.4 数据质量要求

| 检查项 | 阈值（建议） | 处理方式 |
|-------|------------|---------|
| 主键唯一性 | `user_id` + `report_date` 无重复 | 去重并记录 |
| 缺失率 | 单字段缺失 > 80% 需评审是否入模 | 标记或剔除 |
| 时间逻辑 | 开立日期 ≤ 报告日期；逾期天数 ≥ 0 | 异常值修正或剔除 |
| 样本量 | 训练集 ≥ 10,000（视业务调整） | 不足时延长取样窗口 |
| 标签比例 | 正负样本比记录并处理不平衡 | SMOTE / 权重 / 欠采样 |

### 3.5 数据安全与合规

- **严禁**将含真实个人信息的原始数据提交至公开仓库
- 本地开发使用脱敏/抽样数据；`data/` 目录已加入 `.gitignore`
- 所有特征产出需可追溯至原始字段（数据血缘）
- 遵守《个人信息保护法》及公司内部数据使用规范

## 4. 特征工程规划

### 4.1 特征分层架构

```
原始征信字段
    ↓ 清洗 & 标准化
一级特征（直接映射）
    ↓ 聚合 & 统计
二级特征（账户级 → 用户级）
    ↓ 时序 & 交叉
三级特征（趋势/比率/组合）
    ↓ 筛选 & 编码
模型输入特征矩阵
```

### 4.2 特征域规划

| 特征域 | 描述 | 示例子特征 |
|-------|------|-----------|
| **F01-人口统计** | 基本信息衍生 | 年龄段、工龄估算、城市等级 |
| **F02-负债结构** | 各类贷款余额与占比 | 总负债、信用卡使用率、房贷占比 |
| **F03-还款行为** | 历史还款与逾期 | 历史最大逾期天数、近12月逾期次数、逾期严重度 |
| **F04-账户活跃度** | 账户开立与结清 | 活跃账户数、平均账龄、近6月新开账户数 |
| **F05-查询行为** | 硬查询/软查询 | 近3月查询次数、查询机构多样性、查询频率趋势 |
| **F06-授信能力** | 额度与使用率 | 总授信额度、额度使用率均值/最大值 |
| **F07-时序趋势** | 滑动窗口统计 | 近3/6/12月负债变化率、逾期趋势 |
| **F08-交叉组合** | 多域交叉 | 查询次数/账户数、逾期次数×逾期金额 |
| **F09-公共信息** | 负面公共记录 | 是否有判决/强制执行、负面记录计数 |

### 4.3 特征命名规范

```
{域代码}_{聚合方式}_{时间窗口}_{原始字段}

示例：
  F03_max_overdue_days_12m     # 还款行为域，近12月最大逾期天数
  F05_count_query_org_3m       # 查询行为域，近3月查询机构数（去重）
  F02_ratio_cc_balance_total   # 负债结构域，信用卡余额占总负债比
```

### 4.4 特征筛选标准

| 阶段 | 方法 | 剔除条件 |
|-----|------|---------|
| 单变量分析 | IV 值、缺失率 | IV < 0.02 且业务无解释力 |
| 相关性 | Pearson/Spearman | \|r\| > 0.95 的冗余特征保留 IV 更高者 |
| 共线性 | VIF | VIF > 10（逻辑回归场景） |
| 稳定性 | PSI | PSI > 0.25 触发告警与复评 |
| 模型驱动 | 特征重要性 / SHAP | 保留 Top-K 或累计贡献 95% |

### 5.3 建模评估指标

| 任务类型 | 主要指标 | 辅助指标 |
|---------|---------|---------|
| 二分类（需求意向） | AUC-ROC, KS | Precision, Recall, F1, Lift |
| 多分类（需求品类） | Macro-F1, AUC-OVR | 混淆矩阵, 各类 Recall |
| 回归（额度需求） | MAE, RMSE | R², 分位数误差 |

**稳定性监控**：训练集 vs 测试集/OOT 集的 PSI < 0.1（理想）/ 0.25（警戒线）

## 7. 风险与假设

### 7.1 关键假设

- 可获得足够样本量的历史征信数据与业务标签
- 征信报告解析结果字段稳定，版本变更会提前通知
- 业务方能在项目早期明确预测目标与标签定义

### 7.2 已知风险

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| 标签噪声大 | 模型效果上限低 | 多源标签交叉验证；延长观察窗口 |
| 样本不平衡 | 模型偏向多数类 | 重采样、class_weight、调整阈值 |
| 征信字段变更 | 特征流水线断裂 | 配置化字段映射；版本化 schema |
| 数据泄露 | 评估指标虚高 | 严格时间切分；特征仅用报告日前的信息 |
