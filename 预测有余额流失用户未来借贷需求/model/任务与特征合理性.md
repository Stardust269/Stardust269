# 建模任务说明：特征与标签是否匹配

## 业务问题（准确表述）

**不是**「预测用户会不会在马消复贷提现」，而是：

> 在锚点日 `days_dt`，对「有余额、近 60 天未提现、额度≥2万」的用户，预测其在未来 **60 天内外部征信循环贷余额是否会上升**（`zx_balance_label`）。

- 标签看的是 **剔马消** 后的征信余额变化（`org_manage_code <> T10156530H0001`）
- 正样本 ≈ 在外部机构出现 **加杠杆 / 再借贷** 信号

因此这是 **外部多头借贷需求** 预测，不是单纯的站内提现预测。

---

## 结论：征信 + 马消 联合建模 **合理**，但角色不同

| 特征族 | 是否合理 | 理由 |
|--------|----------|------|
| **征信特征** | ✅ 核心、强相关 | 标签本身由征信余额定义；额度利用率、硬查询、逾期、信用卡、房贷/公积金资质等，直接刻画 **外部负债能力与借贷意愿** |
| **马消特征** | ✅ 补充、合理 | 标签虽为「外部」变化，但用户是马消存量有余额客户；站内余额/额度走势、历史提现频次与通过率，反映 **资金压力与借贷习惯**，对「是否会转向外部加杠杆」有增量信息 |
| **cohort 筛选字段** | ❌ 不应入模 | `had_0_30_zx` 等在五条件下全样本恒定，无区分度 |
| **前向窗口统计** | ❌ 不应入模 | `zx_report_cnt_fwd_60d` 与标签共用锚点后 60 天，属于未来信息 |

### 为什么马消特征不算「用错标签」

1. Cohort 已排除 **近 60 天马消提现**，避免「刚提现 → 外部余额变」的短期混淆；马消特征主要是 **锚点前的行为与状态**。
2. 业务上常关心：存量客户是否会 **去外部借钱**；站内行为（额度吃满、申请习惯）是合理先导信号。
3. 若产品目标是 **站内复贷营销**，应另建标签（如锚点后是否在马消提现成功），那是另一个任务。

### 征信特征为什么必须用

标签 = 外部征信余额轨迹 → 不用征信特征等于不用最直接的可观测变量。查询类、循环贷利用率、逾期等均为经典风险/需求信号。

## 缺失值处理是否合理

| 做法 | 评价 |
|------|------|
| 标签缺失 → 删行 | ✅ 无法监督学习 |
| 特征缺失 → 不删行、不填均值 | ✅ 征信稀疏常见；LightGBM 原生支持 NaN 分裂 |
| 类别缺失 → `__MISSING__` | ✅ 与「无单位类型/无月份信息」区分 |
| 不用 forward 窗口特征 | ✅ 已剔除 `zx_report_cnt_fwd_60d` |

**注意（SQL 层，上线打分需对齐）**：宽表里 `latest_*` 在锚点前无报告时，可能用锚点后 60 天内报告兜底。训练集可接受；**线上推理只能使用 `days_dt` 当日及以前的征信**，否则会有时间泄漏。后续可在 SQL Part 5 收紧为仅 `days_dt_zx <= days_dt`。
