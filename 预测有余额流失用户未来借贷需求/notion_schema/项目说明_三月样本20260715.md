## 二、同事三句话解读

1. **「你可以基于我的这个表去加工你的 label 表」**  
   样本底池用同事 `yye_pril_bal_info_20260715_2` 同口径；我方在 `jcr_pril_bal_info_20260715` 自建等价逻辑，再叠加征信余额标签 `zx_balance_label`。

2. **「叠加后面征信余额的增加与减少」**  
   我方 Step 7：样本日 `days_dt` 起 **forward 60 天**内，外部征信循环贷余额（剔马消 `T10156530H0001`）最早值 vs 最大值 → `zx_balance_label`。

3. **「with_0_30+with_31_60=0 and no_balance_flg_60=1；这样你就是我的子集。我做的马消特征你都可以用」**  
   - cohort：`crdt_lim_yx>=20000` + 上述条件 → `jcr_cohort_20260715`  
   - 粒度：**uuid + dt**（同用户三月可能有多条样本）  
   - 马消表只读关联：`ayh_feature_pril_bal_crdt_lim_yx`、`ayh_feature_wdraw_fq_suc`

## 三、样本定义

### 3.1 入模样本池（Step 0a）

```sql
from lj_iceberg.ayh_mkt.ayh_mkt_yx_cust_type_base_df
where dt between '20250801' and '20251031'
  and sx_rowid = 1
  and prod_cd = '5103'
  and if_lend = '复贷'
  and cust_types_01 = '有余额'
  and crdt_lim_yx > 0
```

- `rk=1`：每月内额度利用率**最低**的那一天
- `m = substr(dt,1,6)`：样本月份（202508/202509/202510）

### 3.2 分析 cohort（约 1.5w）

| 条件 | 说明 |
|------|------|
| `crdt_lim_yx >= 20000` | 有效额度 ≥ 2 万 |
| `no_balance_flg_60 = 1` | 样本日后 60 天内变为无余额 |
| `with_0_30 + with_31_60 = 0` | 前 60 天无提现 |

**核验 SQL**（同事口径）：

```sql
select m, count(1), count(distinct uuid) as usr_num,
       sum(if(with_0_30+with_31_60=0 and no_balance_flg_60=1, 1, 0)) as pnum
from lj_iceberg.ai_decision_dev.yye_pril_bal_info_20260715_2
where crdt_lim_yx >= 20000
group by m;
```

我方等价：`select count(1) from jcr_cohort_20260715`

## 四、标签定义

| 字段 | 含义 |
|------|------|
| `zx_balance_label` | 征信余额是否增加：1=有需求，0=无需求，null=观察窗无征信 |
| `mx_cohort_label` | 同事马消侧 label（`with_0_30+31_60=0 & no_balance_60`） |
| `colleague_cohort_flg` | 与 cohort 条件一致的 0/1 标记 |
| `cohort_eligible` | uuid+dt 在 `jcr_cohort_20260715` 中 |

标签窗：`days_dt` ~ `days_dt + 60`（**非** days_dt_1）
