## 一、三步漏斗（与同事口径一致）

| 步骤 | 条件 | 预期人数（十月单月） |
|------|------|----------------------|
| **① 入口** | 5103 有余额 + `crdt_lim_yx>=20000` + 月内 rk=1 | ~5,022,793 |
| **② 流失+未贷** | 60 天内**全渠道**变无余额 **且** **5103** 变无余额 **且** 全渠道未提现 | ~386,311 |
| **③ 征信 cohort** | `had_0_30_zx=1` 且 `had_31_60_zx=1`（锚点 days_dt_1） | **~5,401** |

### 字段对照

| 同事概念 | 我方字段 | 说明 |
|----------|----------|------|
| 全渠道无余额 | `no_balance_flg_60` | 用户-日所有产品均无「复贷+有余额」，且出现「复贷+无余额」 |
| 5103 无余额 | `no_balance_flg_60_5103` | 5103 渠道出现「复贷+无余额」 |
| 全渠道未提现 | `with_0_30 + with_31_60 = 0` | 锚点 days_dt_1 后 0~60 天无提现（**不限 5103**） |
| 5103 未提现（可选） | `with_0_30_5103 + with_31_60_5103 = 0` | 同事说「可以加条件选的」 |
| 征信 0~30 / 31~60 | `had_0_30_zx` / `had_31_60_zx` | 锚点 days_dt_1 |

### 锚点

- `no_balance`：锚点 **days_dt**（最低利用率日）
- `had` / `with` / 征信特征 / 标签：锚点 **days_dt_1** = `date_sub(days_dt, 1)`
