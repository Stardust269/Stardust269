# 项目说明：标签构建（Label）

## 一、业务目标

预测 **当前有效额度 ≥ 2 万元且有余额** 的用户，在样本日后是否会产生 **外部征信余额增加**（有再借贷/加杠杆需求）。

---

## 二、样本定义

### 2.1 入模样本池（训练标签 cohort）

| 项目 | 说明 |
|------|------|
| **时间** | **2025 年 8 月、9 月、10 月**（`202508` / `202509` / `202510`） |
| **用户条件** | 当月 **当前有效额度 ≥ 20,000** 且 **有余额** |
| **产品渠道** | **全渠道** |
| **样本日 `days_dt_1`** | 当月内 **额度利用率最低** 的那一天（「最低额度日」） |
| **观察窗** | `days_dt_1` 之后 **0～60 日** |

#### 10 月宽口径底池（业务文档）

在 **2025-10-01～10-31** 期间，**当前有效额度 ≥ 2 万**、**全渠道**用户：

| 指标 | 业务文档人数 | 当前样本表复现 |
|------|-------------|----------------|
| 10 月宽口径底池 | **5,022,793** | **4,196,692**（`jcr_pril_bal_info` 10 月） |

两者相差约 **82.6 万**，复现 5,401 时需确认 `yye_pril_bal_info` 是否为同一「全渠道」全量。

### 2.2 纳入标签统计的过滤条件（精确口径）

**时间锚点**：`days_dt_1` = 当月内 **额度利用率最低** 的那一天（「月内最低额度日」）。  
**观察窗**：`days_dt_1` 之后 **0～60 日**（与 `label_fwd_days = 60` 一致）。

在该窗口内，**同时满足**：

| # | 业务条件 | 样本表字段（SQL） | 说明 |
|---|----------|-------------------|------|
| 1 | **转为无余额** | `no_balance_flg_60 = 1` | 0～60 日内由有余额变为无余额；**不是** `no_balance_flg_90` |
| 2 | **未发起贷款** | `with_0_30 + with_31_60 = 0` | 0～30、31～60 两段均未发起贷款；**不含** `with_61_90` |
| 3 | **有征信报告** | `had_0_30_zx = 1` **且** `had_31_60_zx = 1` | **第 0～30 日有征信，且第 31～60 日也有征信**（两段 AND，不是 OR） |

业务文档（10 月）原文：「最低额度后 0～60 日，第 0～30 日有征信、第 31～60 日有征信的用户一共 **5,401** 个」——与第 3 条 **AND** 口径一致。第 1、2 条为标签统计附加条件（转为无余额、未发起贷款）。

满足上述条件的用户才进入 **5,401 / 1.5 万** 的标签统计池；再按征信余额是否增加打 `label`。

**10 月标签统计池**：

| 指标 | 业务文档 | 当前 SQL 复现 |
|------|----------|---------------|
| 纳入标签统计（合计） | **5,401** | **4,797**（差 604，约 11%） |
| 无需求 `label = 0` | **3,511** | 3,112 |
| 有需求 `label = 1` | **1,890** | 1,685 |
| 正样本占比 | **35.0%** | **35.1%**（比例一致，label 逻辑基本正确） |

**8～10 月合计约 1.5 万**（各月分别按上表口径统计后相加）。

> 占总体的比例约 **0.11%**（高度稀疏场景）。

#### 人数差异说明（5401 vs 4797）

| 可能原因 | 说明 |
|----------|------|
| **底池不一致** | 文档 502 万 vs 样本表 419 万，少约 82.6 万用户 |
| **样本表渠道** | 文档为「全渠道」，复制的 `yye` 表可能是子集 |
| **label 非空** | SQL 要求 `label is not null`，业务侧可能先圈 5401 再打标 |

#### 常见误用（会导致人数偏少，例如 10 月仅 457）

| 错误写法 | 正确写法 |
|----------|----------|
| `no_balance_flg_90 = 1` | `no_balance_flg_60 = 1` |
| `had_61_90_zx = 1` | 不需要；0～60 只需 `had_0_30_zx` **且** `had_31_60_zx` |
| `with_0_30 + with_31_60 + with_61_90 = 0` | `with_0_30 + with_31_60 = 0` |
| `(had_0_30_zx=1 or had_31_60_zx=1)` | 业务要求两段都有，必须用 **AND** |

#### 核验 SQL（10 月）

```sql
-- 宽口径底池（应对齐文档 5,022,793）
select count(1) as base_cnt
from lj_iceberg.ai_decision_dev.jcr_credit_feature_label_20260623
where sample_month = '202510' and crdt_lim_yx >= 20000;

-- 标签统计池（应对齐文档 5,401 / 3511 / 1890；当前复现 4797 / 3112 / 1685）
select
    count(1) as cohort_cnt,
    sum(case when label = 0 then 1 else 0 end) as neg_cnt,
    sum(case when label = 1 then 1 else 0 end) as pos_cnt
from lj_iceberg.ai_decision_dev.jcr_credit_feature_label_20260623
where sample_month = '202510'
  and dataset_split in ('train', 'val')
  and no_balance_flg_60 = 1
  and had_0_30_zx = 1
  and had_31_60_zx = 1
  and with_0_30 + with_31_60 = 0
  and label is not null;
```

## 三、标签定义

| 标签值 | 含义 | 判定逻辑（征信侧） |
|--------|------|-------------------|
| **正样本 `label = 1`** | 有需求 | 样本日后 60 天内，**征信余额有增加** |
| **负样本 `label = 0`** | 无需求 | 样本日后 60 天内，**征信余额未增加** |

### 征信余额是否增加（与 SQL 对齐）

在样本日 `days_dt_1` 后 **60 天**内关联到的征信报告中：

- `first_balance`：窗口内 **最早一份** 征信的非马消循环贷总余额
- `balance_max`：窗口内 **最大** 非马消循环贷总余额
- **label = 1** 当且仅当 `balance_max > first_balance`

## 四、数据集划分

| 数据集 | 时间/范围 | 用途 |
|--------|-----------|------|
| **训练集 + 验证集** | 2025-08 / 09 / 10 标签 cohort | 模型训练；建议按用户 `uuid` 做 train/val 切分（如 80/20） |
| **测试集（效果评估）** | **2025-11-01** | 离线评估模型效果；当日有效额度 ≥2W 且有余额用户，用 **当日可获取特征** 预估 |

### 预测场景（线上/回测）

- **打分对象**：当前有效额度 ≥ 2W 且有余额的用户
- **特征时点**：**当日**能获取到的特征（观察日 = 当天）
- **测试日**：**2025 年 11 月 1 日**
