# 项目说明：标签构建（Label）

## 一、业务目标

预测 **当前有效额度 ≥ 2 万元且有余额** 的用户，在样本日后是否会产生 **外部征信余额增加**（有再借贷/加杠杆需求）。

---

## 二、样本定义

### 2.1 入模样本池

| 项目 | 说明 |
|------|------|
| **时间** | 2025 年 10 月（`dt` 20251002~20251101，分区延后 1 天） |
| **用户条件** | `prod_cd=5103` 复贷有余额，`rk=1` 月内最低额度利用率日 |
| **`days_dt_1`** | `date_sub(days_dt, 1)`（had/with/label 锚点） |

### 2.2 分析 cohort（同事最后一查，**5401**）

**`cohort_eligible = 1` 条件**（Step 5/7，2026-07 同事确认版）：

```sql
crdt_lim_yx >= 20000
and had_0_30_zx = 1 and had_31_60_zx = 1
and no_balance_flg_60 = 1
and with_0_30 + with_31_60 = 0
```

| 条件 | 说明 |
|------|------|
| `had_0_30` + `had_31_60` | **仅 2 段征信**（不含 61~90、91~120） |
| `no_balance_flg_60` | 60 天内变为无余额（锚点 `days_dt`） |
| `with_0_30 + with_31_60 = 0` | 60 天内无提现（锚点 `days_dt_1`） |

**核验 SQL**：

```sql
select count(1) as cohort_cnt
from lj_iceberg.ai_decision_dev.jcr_credit_feature_label_20260623
where cohort_eligible = 1;
```

## 三、标签定义

| 标签值 | 含义 | 判定逻辑 |
|--------|------|----------|
| **正样本 `label = 1`** | 有需求 | `fwd_max_balance > fwd_first_balance` |
| **负样本 `label = 0`** | 无需求 | forward 60 天余额未增加 |

- `fwd_first_balance`：`days_dt_1~+60` 内最早征信日（`zx_rank=1`）非马消循环贷余额合计
- `fwd_max_balance`：同窗口内最大余额合计
- 剔马消：`org_manage_code <> 'T10156530H0001'`
